# Script Instructions

## File & structure

- Primary language for this workspace: **{{PRIMARY_LANGUAGE}}**
- Package manager: **{{PACKAGE_MANAGER}}**
- Keep scripts under ~150 LOC; one top-level responsibility per file.
- Scripts that are executable must have a shebang line appropriate to `{{PRIMARY_LANGUAGE}}` and be marked executable in the repo.
- Generated or derived artifacts must be regenerated by a script in `scripts/` whose name reflects the artifact it produces, not edited by hand.

## Dependencies & paths

- No silent error swallowing: fail loudly with a non-zero exit code or a structured error.
- Prefer stdlib or built-in tooling over introducing new dependencies.
- When a third-party dependency is unavoidable, declare it with a pinned version and document why stdlib was insufficient.
- Do not hard-code paths that vary by environment — use environment variables or config.
